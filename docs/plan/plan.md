## 実装方針

- フロント（Electron + React + TypeScript + TipTap）とバックエンド（FastAPI）を HTTP (localhost) で分離する。
- LLM 呼び出しはすべて OpenAI 互換 `/v1/chat/completions`。執筆・校正はストリーミング。
- 文書の正は TipTap JSON（`document.content_json`）。Markdown は書き出し用の派生。
- 埋め込みは Ruri、検索は hybrid（sqlite-vec + FTS5）固定。
- LangChain 等の重いフレームワークは使わない。
- 各フェーズの終わりで「動くもの」を維持する（フェーズ 1 完了時点で LLM なしのエディタとして使える状態にする）。

- [x] 書式ツールバー: 太字 / 斜体 / 下線 / 打ち消し / インラインコード / 見出し H1-H3 / 箇条書き / 番号リスト / 引用 / コードブロック / 水平線 / リンク（挿入・解除）/ undo・redo。アクティブ状態表示、ショートカット併用可
- [x] 表（テーブル）ツール: `@tiptap/extension-table`（v3, resizable）導入。書式バーの「表」ボタンで挿入、表内カーソル時に文脈ツールバー（行/列 追加削除・結合/分割・ヘッダ行・削除）。GFM 相互変換は tiptap-markdown が担当（読み込み: markdown-it→HTML→parseHTML / 書き出し: 組み込みの table シリアライザ。v3 の native renderMarkdown は tiptap-markdown 経路では未使用）。**制約**: セル結合・複数段落セルは HTML 化 → `html:false` のため書き出しで落ちる。ヘッダ行必須。
- [ ] **Markdown 形式のコピペが上手くいかない問題（2026-07-08 ユーザー報告）**: 語句をコピーして貼り付けると、貼り付け位置の前後に空段落（余計な改行が2段ほど）ができる。`transformPasted` で空段落除去＋単一段落のインライン化を試みたが直らず、いったん revert 済み。要再調査（コピー元＝ブラウザ / チャットパネル / PDF / エディタ自身 等でスライス構造が異なる可能性。実アプリで貼り付け直後の ProseMirror スライスを観察して原因を特定する）。`transformPastedText: true`（tiptap-markdown の Markdown 貼り付け）は維持する前提。
