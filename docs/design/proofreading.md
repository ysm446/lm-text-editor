# 校正（プルーフリード）ガイド

作成日時: 2026-07-08 12:45
更新日時: 2026-07-08 12:45

校正機能の仕組みと、システムプロンプトによる挙動のカスタマイズ方法をまとめる。
実装は `backend/llm/prompts.py`（プロンプト）、`backend/main.py`（`/review/*`）、`src/review/`（UI）。

## 1. 校正の二モード

- **インライン校正（選択範囲）** — 選択したテキストを LLM で校正し、同一画面で diff（赤=削除 / 緑=追加）表示。採用 / 破棄。`build_review_messages` に選択範囲 + 前後文脈を渡す。
- **分割ビュー校正（文書全体）** — 段落単位で校正し、左右分割で before/after を比較。段落ごと採用 / 破棄・一括採用。

どちらも LLM への指示は同じ**校正システムプロンプト**を使う。前後の文脈は「参考」として渡すだけで、出力には含めない。

## 2. システムプロンプトの構造

- 既定値は `backend/llm/prompts.py` の `REVIEW_SYSTEM`。
- 設定（⚙️ → **プロンプト**）で上書きできる。上書きは `settings.review_system_prompt` に保存され、`prompts.review_system()` が「上書きがあればそれ、空なら既定」を返す。
- 空にして保存 / 「既定に戻す」で既定へ復帰。バックエンド再起動なしで次回の校正から反映（校正のたびに設定を読む）。

既定プロンプトの要点:

- 意味と事実関係を変えず、日本語として自然で読みやすくする。
- 校正後の本文のみを出力（説明・前置き・引用符なし）。
- Markdown・改行・コード・URL・専門用語は維持する。
- **修正が不要な箇所は変更しない。**
- 前後の文脈は参考であり出力に含めない。

## 3. 文体・語尾を変える（敬体 ↔ 常体）

文末の「です・ます（敬体）」↔「である・だ（常体）」の統一は、**意味を変えない純粋な文体変換**なのでプロンプトで確実に実現できる。

ポイントは、既定の「**修正が不要な箇所は変更しない**」を上書きすること。文体がすでに一貫している文は「修正不要」と判断されて据え置かれるため、「文体をそろえる書き換えは必須」と明示する。

### 書き方のコツ

1. どの文体に揃えるかを断定形で書く（「常体に統一」「敬体に統一」）。
2. 「文体をそろえる書き換えは必須とし、修正不要とみなさない」と書いて既定ルールを上書きする。
3. 文体以外は変えない（意味・事実・専門用語・Markdown・コード・URL）と釘を刺す。
4. 「記事全体で統一」と一貫性を求める。

### 例: 常体（である・だ調）に統一

設定 → プロンプト → 校正のシステムプロンプトに貼り付ける。

```
あなたは技術ブログ記事の校正者です。与えられた「校正対象」を、日本語として自然で読みやすく校正してください。

文体ルール:
- 文末は常体（「である・だ」調）に統一する。敬体（「です・ます」調）の文はすべて常体に書き換える。
- 文体をそろえる書き換えは必須とし、「修正不要」とはみなさない。
- 文体以外は意味・事実・専門用語・Markdown 記法・改行・コード・URL を変えない。

出力ルール:
- 校正後の本文のみを出力する。説明・前置き・引用符は一切書かない。
- 前後の文脈は参考情報であり、出力に含めない。
```

### 例: 敬体（です・ます調）に統一

上の「常体」「敬体」を入れ替えるだけ（「文末は敬体（です・ます調）に統一する。常体の文はすべて敬体に書き換える。」）。

## 4. 時制（「る」↔「した」）を変える場合の注意

「動作する」↔「動作した」のような現在形 ↔ 過去形の変更は、**文体ではなく時制**であり、**意味・事実の含みが変わる**ことがある（過去の実測 vs 一般的な性質）。

- 「意味を変えない」と「時制を変える」は衝突しやすく、LLM が過剰に書き換えたり遠慮したりしやすい。
- 揃えたい場合は**範囲と条件を具体的に**指定する。例:
  ```
  - 手順や解説の地の文は現在形（「〜する」）で統一する。ただし過去の実測結果・実際に起きた出来事を述べた文の時制は変えない（意味が変わるため）。
  ```
- 敬体/常体ほど機械的でないので、diff を確認しながら使う。

## 5. 使い方・留意点

- 文体変換は多くの文が変わるため diff は大きくなる。**選択範囲を校正**は選んだ箇所だけ、**全体を校正**は段落ごとに適用される。
- 効きが弱いときはプロンプトの文言を強める（「例外なく」「1 文ずつ確認して」等）。
- 校正だけでなく**執筆（続き生成・セクション生成）の文体**も揃えたい場合、それらは別プロンプト（`CONTINUE_SYSTEM` / `SECTION_SYSTEM`）。現状は設定で編集できるのは校正のみ。必要になれば執筆側も編集可能にする（プロンプト編集 UI を拡張）。
