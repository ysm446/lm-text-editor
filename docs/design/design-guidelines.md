# UI デザインガイドライン

作成日時: 2026-07-07 14:10
更新日時: 2026-07-07 14:25

本アプリの UI を設計・変更するときの指針。**新しい UI はこのガイドラインに照らして作り、外れる場合は先にこのファイルを更新してから実装する。**
トークンの実体は `src/styles.css` 冒頭の `:root` / `[data-theme='dark']` ブロック。

## 1. 基本方針 — 「静かなクローム」

- **本文（紙）が主役**。周辺 UI（バー・サイドバー・ツールバー・パレット）は小さく・淡く・枠線なしを基本にする。
- 領域は**枠線ではなく面の差**で区切る。紙 = `--surface`、クローム = `--bg`。枠線（`--border`）を使うのは入力欄・パネル・モーダルの輪郭だけ。
- 装飾は追加しない。色・影・動きは以下で定義されたものだけを使う。

## 2. 色

**色は必ずトークン経由で指定する。HEX 直書き禁止**（例外: 資料種別バッジなどのカテゴリ色）。
ライト/ダークは `[data-theme]` でトークンごと切り替わるため、コンポーネント側でテーマ分岐を書かない。

| 役割 | トークン | 用途 |
|---|---|---|
| アクセント（唯一の主張色・青） | `--accent` / `--accent-text` | 主ボタン、選択中、アクティブ、フォーカスリング、リンク |
| アクセント淡色 | `--selected` | 選択行・アクティブトグルの背景 |
| 成功・稼働 | `--ok` / `--ok-bg` | モデル稼働ドット、採用済み行 |
| 注意 | `--warn` / `--warn-bg` | 未保存表示、ドラフト復元バナー |
| 危険・削除 | `--danger` / `--danger-bg` | 削除メニュー、エラー面 |
| diff | `--del-bg/--del-text` / `--ins-bg/--ins-text` | 校正・履歴の削除/追加 |
| 面と文字 | `--bg` `--surface` `--text` `--muted` `--faint` `--border` `--hover` | 上表以外のすべて |
| 上下バー専用（テーマ非依存ダーク） | `--bar-bg` `--bar-surface` `--bar-hover` `--bar-border` `--bar-text` `--bar-muted` `--bar-faint` `--bar-ok` `--bar-warn` `--bar-danger` | 上部バー・ステータスバー・トースト |

原則: **アクセントは青 1 色**。「目立たせたい」ときに新しい色を持ち込まない。緑・黄・赤は状態の意味（成功/注意/危険）にのみ使う。

## 3. 寸法・タイポグラフィ

| トークン | 値 | 用途 |
|---|---|---|
| `--radius-s` | 4px | アイコンボタン・バッジ・インラインコード |
| `--radius-m` | 6px | ボタン・入力・メニュー項目 |
| `--radius-l` | 10px | パネル・モーダル・カード |
| `--fs-xs` | 11px | ラベル・バッジ・ステータスバー・グリップ |
| `--fs-sm` | 12px | メニュー・パネルヘッダ・補足・バー内 |
| `--fs-ui` | 13px | 標準 UI（ボタン・サイドバー・入力） |
| `--fs-md` | 14px | パネル本文（diff・要約など読ませる文章） |

- UI の文字サイズは **px 固定**（em の入れ子で実サイズが揺れるのを防ぐ）。この 4 段階以外を増やさない。
- 本文だけが設定値 `--editor-font-size` に追従し、本文内の見出し等は em で相対指定する。
- 数値が並ぶ場所（ステータスバー等）は `font-variant-numeric: tabular-nums`。
- 影は `--shadow-1`（面の浮き・パネル）と `--shadow-2`（メニュー・モーダル・パレット）の 2 種のみ。

## 4. コンポーネント規約

新しい UI 部品は、まず以下のどれかに当てはめる（`styles.css` の「共通コンポーネント」節のセレクタ群に追加する）。

- **ボタン**
  - 枠付き ghost（既定）: パネル・ダイアログ内の操作。`--border` 枠 + hover で `--hover`。
  - primary: `--accent` 塗り + 白文字 + 太字。**1 画面（1 パネル）に 1 つまで**。その画面の「主目的」だけ（保存・採用・取り込む等）。
  - メニュー項目: borderless・全幅・左寄せ。選択中は `--selected` + `--accent` 文字。
  - アイコンボタン: borderless・`--radius-s`・`--muted` 文字。
  - バー内 chip: `--bar-surface` 塗り・枠なし。
- **パネル**（本文上に出るカード: 校正結果・履歴など）: `--border` 枠 + `--radius-l` + `--shadow-1`。ヘッダは 1 行（ラベル `--fs-sm` `--muted`、操作は右寄せ）。本文は max-height + 内部スクロール。
- **モーダル**: `--radius-l` + `--shadow-2`、`--overlay` の上。ヘッダ構造はパネルと同じ。
- **フローティングパレット**: `ToolPalette` コンポーネントを使う。`position: fixed`、`--shadow-2`、グリップでドラッグ移動、位置は localStorage に記憶。z-index は 90（sticky ツールバー 50 < パレット 90 < モーダル 100+ < ライトボックス 300 < トースト 400）。
- **入力欄**: `--border` 枠 + `--radius-m`、focus で `--accent` 枠。

## 5. レイアウト

- 並びは **ツールバー（sticky）→ タイトル → 本文**。操作ツールバー（校正・執筆支援・履歴・保存）は全幅・上部固定、書式ツール（太字・見出し・リスト等）はフローティングパレット。
- 本文とタイトルは中央 **768px カラム**（左右 64px インセット）。本文上に出るバナー・パネルは 720px 幅で中央揃え。
- サイドバー = `--bg`（クローム）、本文ステージ = `--surface`（紙）のフルブリード。カード枠は作らない。
- 上下バーはテーマ非依存のダーク（VS Code 方式）。

## 6. インタラクション

- hover/active の遷移は `0.12s ease`（背景・文字色のみ）。それより長い・大きいアニメーションはスライドペイン（0.28s）とトーストだけ。
- `prefers-reduced-motion: reduce` で全アニメーションを止める（styles.css 末尾で一括処理済み）。
- キーボードフォーカスは共通のアクセントリング（`:focus-visible`）。上部バー内のみ例外で無効。
- クリックできるものは hover で必ず反応を返す（背景 `--hover` など）。

## 7. 変更時のチェックリスト

- [ ] 色・角丸・文字サイズ・影はすべてトークン参照か（HEX / px 直書きを増やしていないか）
- [ ] ボタンは 5 系統のどれかに乗っているか。primary を増やしすぎていないか（1 画面 1 つ）
- [ ] ライト/ダーク両テーマで確認したか
- [ ] スクロール位置に関係なく操作と結果が見えるか（sticky / パレット / パネルの配置）
- [ ] フォーカスリング・disabled・hover の状態があるか
- [ ] このガイドラインと矛盾する場合、先に本ファイルを更新したか
