# UI デザインガイドライン

作成日時: 2026-07-07 14:10
更新日時: 2026-07-08 11:45

本アプリの UI を設計・変更するときの指針。**新しい UI はこのガイドラインに照らして作り、外れる場合は先にこのファイルを更新してから実装する。**
トークンの実体は `src/styles.css` 冒頭の `:root` / `[data-theme='dark']` ブロック。

## 1. 基本方針 — 「静かなクローム」

- **本文（紙）が主役**。周辺 UI（バー・サイドバー・ツールバー・パレット）は小さく・淡く・枠線なしを基本にする。
- 領域は**枠線ではなく面の差**で区切る。紙 = `--surface`、クローム = `--bg`。枠線（`--border`）を使うのは入力欄・パネル・モーダルの輪郭だけ。
- 装飾は追加しない。色・影・動きは以下で定義されたものだけを使う。
- 参照デザインは **Obsidian**（配色・スクロールバー・リサイズ・Vault スイッチャー等）。迷ったら Obsidian の挙動に寄せる。

## 2. 色

**ダークテーマが基準**。配色は Obsidian のデフォルトダークに寄せる（紙 #1e1e1e / クローム #191919 / 文字 #dadada / アクセントは紫 #7f6df2）。ライトは同じ紫を軸にした従属テーマで、デザイン検討はまずダークで行う。

**色は必ずトークン経由で指定する。HEX 直書き禁止**（例外: 資料種別バッジなどのカテゴリ色）。
ライト/ダークは `[data-theme]` でトークンごと切り替わるため、コンポーネント側でテーマ分岐を書かない。

| 役割 | トークン | 用途 |
|---|---|---|
| アクセント（唯一の主張色・紫） | `--accent` / `--accent-text` | 主ボタン、アクティブ状態、フォーカスリング、リンク、モデルバーのロード中 |
| 選択ハイライト | `--selected` | 選択行・アクティブトグルの背景（ダークではニュートラルなグレー。色では主張しない） |
| 成功・稼働 | `--ok` / `--ok-bg` | モデル稼働ドット、採用済み行 |
| 注意 | `--warn` / `--warn-bg` | 未保存表示、ドラフト復元バナー |
| 危険・削除 | `--danger` / `--danger-bg` | 削除メニュー、エラー面 |
| diff | `--del-bg/--del-text` / `--ins-bg/--ins-text` | 校正・履歴の削除/追加 |
| 面と文字 | `--bg` `--surface` `--text` `--muted` `--faint` `--border` `--hover` | 上表以外のすべて |
| スクロールバー | `--scrollbar-thumb` / `--scrollbar-thumb-hover` | つまみの色（トラックは透明） |
| 上下バー専用（テーマ非依存ダーク） | `--bar-bg` `--bar-surface` `--bar-hover` `--bar-border` `--bar-text` `--bar-muted` `--bar-faint` `--bar-ok` `--bar-warn` `--bar-danger` | 上部バー・ステータスバー・モデルバー・トースト |

原則: **アクセントは紫 1 色**。「目立たせたい」ときに新しい色を持ち込まない。緑・黄・赤は状態の意味（成功/注意/危険）にのみ使う。
カテゴリ色（資料バッジ）は例外的に固定 HEX: 手動メモ=紫（アクセント）、ファイル/リファレンス=緑、記事=紫系、Web=グレー（`#8a8f98`。青は主張が強すぎるため落とした）。手動メモ（source_type='note'）は編集可能で、本文の正は `manual_note` テーブル、チャンクは `note://<id>` で派生。

## 3. 寸法・タイポグラフィ

| トークン | 値 | 用途 |
|---|---|---|
| `--radius-s` | 4px | アイコンボタン・バッジ・インラインコード |
| `--radius-m` | 6px | ボタン・入力・メニュー項目 |
| `--radius-l` | 10px | パネル・モーダル・カード |
| `--fs-xs` | 11px | ラベル・バッジ・ステータスバー・グリップ |
| `--fs-sm` | 12px | メニュー・パネルヘッダ・補足・バー内 |
| `--fs-ui` | 13px | 標準 UI（ボタン・サイドバー・入力） |
| `--fs-md` | 14px | パネル本文（diff・要約など読ませる文章） |

- UI の文字サイズは **px 固定**（em の入れ子で実サイズが揺れるのを防ぐ）。この 4 段階以外を増やさない。
- 本文だけが設定値 `--editor-font-size` に追従し、本文内の見出し等は em で相対指定する。
- 数値が並ぶ場所（ステータスバー・スライダー値等）は `font-variant-numeric: tabular-nums`。
- 影は `--shadow-1`（面の浮き・パネル）と `--shadow-2`（メニュー・モーダル・パレット）の 2 種のみ。

## 4. コンポーネント規約

新しい UI 部品は、まず以下のどれかに当てはめる（`styles.css` の「共通コンポーネント」節のセレクタ群に追加する）。

- **ボタン（5 系統）**
  - 枠付き ghost（既定）: パネル・ダイアログ内の操作。`--border` 枠 + hover で `--hover`。
  - primary: `--accent` 塗り + 白文字 + 太字。**1 画面（1 パネル）に 1 つまで**。その画面の「主目的」だけ（保存・採用・取り込む・保存してチャンク化 等）。
  - メニュー項目: borderless・全幅・左寄せ。選択中は `--selected`（+ 太字）。
  - アイコンボタン: borderless・`--radius-s`・`--muted` 文字。
  - バー内 chip: `--bar-surface` 塗り・枠なし（上下バー内）。
- **パネル**（本文上に出るカード: 校正結果・履歴・執筆支援など）: `--border` 枠 + `--radius-l` + `--shadow-1`。ヘッダは 1 行（ラベル `--fs-sm` `--muted`、操作は右寄せ）。本文は max-height + 内部スクロール。
- **モーダル**: `--radius-l` + `--shadow-2`、`--overlay` の上。ヘッダ構造はパネルと同じ（例: 設定・資料ビューア・Web 検索・手動資料エディタ `NewSourceEditor`）。
- **ドロップダウンメニュー**: `--surface` + `--border` + `--radius-l` + `--shadow-2`、項目はメニュー項目ボタン。クリック外で閉じる（例: ライブラリメニュー、行の「…」、資料の「＋」追加メニュー）。
- **フローティングパレット**: `ToolPalette` を使う。横長のバー型で、**ビューポートではなく本文ステージに付随**させる（sticky アンカー基準の絶対配置。ペイン開閉でステージが動けば一緒に動く）。`--shadow-2`、グリップでドラッグ移動、位置はステージ相対で localStorage に記憶。z-index は 60。
- **入力欄**: `--border` 枠 + `--radius-m`、focus で `--accent` 枠。
- **スライダー**: 細いトラック（4px・角丸）+ 小さめのつまみ（14px・アクセント色）を webkit 疑似要素で自前描画し、左側は `--fill`（つまみ中心の px）までアクセント色。離散ステップは index ↔ 値の配列で対応。**目盛りはつまみ中心の x 座標に絶対配置**（`justify-content: space-between` はつまみ位置とズレるので使わない）。右端に現在値ボックス。
- **既定値に戻す（リセット）UI**: 既定値を持つ設定コントロールには、リセット操作を用意する。インラインの値コントロール（スライダー等）では、**ゴミ箱アイコン（`.settings-reset-btn`）を現在値表示の左に置き**、押すと既定へ戻す。まとまった編集領域（テキストエリア等）では「既定に戻す」テキストボタンでよい（空保存＝既定、も可）。

z-index の階層: sticky ツールバー 50 < パレット 60 < ドロップダウン 150〜200 < モーダル 100+ < ライトボックス 300 < トースト 400。

## 5. レイアウト

- **上部バー**: 3 カラムグリッド `1fr auto 1fr`。左（現在は空き）/ 中央 = モデルピル / 右 = 設定 ⚙️。テーマ非依存のダーク。
- **モデルピル**: 上部中央の状態ボタン（アイコン + 中央テキスト + シェブロン）。クリックでモデル選択モーダルを開く。状態は色で表現 — ロード済み=アクセント塗り / 未ロード=グレー / ロード中=アクセントのシマー（`.model-pill.on/.off/.loading`）。
- **下部ステータスバー**: 常時表示。リソースメーターは右端に寄せ、いちばん右に表示/非表示トグル。非表示中もバーとトグルは残す（消すと戻せないため）。テーマ非依存のダーク。
- **エディタ**: 並びは **操作ツールバー（sticky）→ タイトル → 本文**。操作ツールバー（校正・執筆支援・履歴・保存）は全幅・上部固定、書式ツール（太字・見出し・リスト等）はフローティングパレット。本文とタイトルは中央 **768px カラム**（左右 64px インセット）、本文上のバナー・パネルは 720px 幅で中央。本文ステージは `--surface`（紙）のフルブリードでカード枠なし。
- **サイドバー**: `--bg`（クローム）。上から順に、ワークスペースツリー（ドキュメント/資料/画像はフォルダのように開閉、状態は localStorage）→ 下部フッターにライブラリ（= Obsidian の Vault 相当）スイッチャー。幅はドラッグで可変。行の hover/選択ハイライトは行全体（`.item-row`）が持ち、「…」等の操作ボタンもその枠内に入れる。
- **可変幅の境界**（サイドバー・分割ペイン）は当たり判定 6px の仕切りにし、見た目は中央 1px の線を hover / ドラッグ時だけアクセント色でフェード表示（Obsidian 流）。ドラッグ中は幅トランジションを切る。幅は px で localStorage に記憶。
- **スクロールバー**は細い・角丸・トラック透明の Obsidian 風。独自にトラックを塗らない。

## 6. インタラクション

- hover/active の遷移は `0.12s ease`（背景・文字色のみ）。それより長い・大きいアニメーションはスライドペイン（0.28s）とトーストだけ。
- `prefers-reduced-motion: reduce` で全アニメーションを止める（styles.css 末尾で一括処理済み）。
- キーボードフォーカスは共通のアクセントリング（`:focus-visible`）。上部バー内のみ例外で無効。
- クリックできるものは hover で必ず反応を返す（背景 `--hover` など）。
- ドラッグ操作（パレット・リサイズ）は Pointer Events + `setPointerCapture`、要素に `touch-action: none`。

## 7. 変更時のチェックリスト

- [ ] 色・角丸・文字サイズ・影はすべてトークン参照か（HEX / px 直書きを増やしていないか）
- [ ] ボタンは 5 系統のどれかに乗っているか。primary を増やしすぎていないか（1 画面 1 つ）
- [ ] ライト/ダーク両テーマで確認したか（基準はダーク）
- [ ] スクロール位置・ペイン開閉に関係なく操作と結果が見えるか（sticky / パレット / パネルの配置）
- [ ] フォーカスリング・disabled・hover の状態があるか
- [ ] このガイドラインと矛盾する場合、先に本ファイルを更新したか
